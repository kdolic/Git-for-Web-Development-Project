    37. Walk me through Lambda's git flow. 
    - Fork the repo
    - Check to make sure name is in front of repo
    - Clone a repo 
    - cd command to where you want to clone forked repo
    - use git clone command 
    - cd into newly created repo
    - use git checkout -b (branch-name) to create branch
    - git add . to add changes to branch
    - git commit -m "message" to commit changes
    - git push to push changes back into the Github repo
    - submit pull request
    - go back to forked project repo and make sure on own branch, select name branch inside of main/master
    - click name branch then click new pull request button and pull onto your branch
    - click dropdown that says base repo and change to forked repo instead of original
    - create pull request

Stretch Questions

    1. What is the difference between an inline element and a block element?
    - An inline element does not start on a new line and it only takes up as much width as necessary.
    - always starts on a new line and takes up the full width available

    2. What happens when an element is positioned absolutely? 
    - stays in the same position, no matter the behavior of the other element's positions. has its own absolute position

    3. How do I make an element take up only the amount of space it needs but also have the ability to give it a width?
    - flexbox, flex will take up the space it needs and can provide a width

    4. Name 3 elements that are display block by default, 2 elements that are display inline by default and 1 element that is display inline-block by default
    - p, h1, div
    - span, a
    - span (possibly ul)

    5. In your own words, explain the box model. What is the "fix" for the box model, in other words, how do we make all elements respect the width we've given them? 
    - used when talking about design and layout of sections/divs within the web page. consists of margin, border, padding, content. A way of wrapping all the html elements within the file.
